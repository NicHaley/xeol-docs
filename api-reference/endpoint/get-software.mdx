---
title: "Get Software Info"
api: "GET https://eol.xeol.com/api/v1/{identifier}"
description: "This endpoint gets a software's support information"
---

### Parameters

<ParamField body="identifier" type="string">
  Software's identifier.
  
  Can be either a `cpe`, `purl`, or `custom`
</ParamField>

### Response

<ResponseField name="name" type="string">
  Name of the matched artifact
</ResponseField>

<ResponseField name="version" type="string">
  Release cycle of the matched artifact
</ResponseField>

<ResponseField name="type" type="string">
  Type of the matched artifact, either `software` or `package` or `os` or `product`
</ResponseField>

<ResponseField name="released" type="datetime">
  Release date of the matched artifact
</ResponseField>

<ResponseField name="lts" type="boolean">
  Whether the matched artifact has `Long Term Support`
</ResponseField>

<ResponseField name="els" type="object[]">
  All availabe support including extended lifecycle support for matched artifact
  
  This will always have at least **one** item.
  <Expandable title="els">
    <ResponseField name="id" type="UUID">
      Unique support ID 
    </ResponseField>

    <ResponseField name="name" type="string">
      Official support name
    </ResponseField>

    <ResponseField name="eol" type="datetime">
      End-of-life date
    </ResponseField>
  </Expandable>
</ResponseField>

<RequestExample>
  ```bash curl
  curl --location --request GET 'https://eol.xeol.com/api/v1/cpe%3A%2Fo%3Aredhat%3Aenterprise_linux%3A6' \
  --header 'Content-Type: application/json' \
  --header 'Authorization: <XEOL_API_KEY>' \
  ```
</RequestExample>
